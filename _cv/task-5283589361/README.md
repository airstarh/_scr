# Замечания к своему решению

- Простите, но мой аккаунт в ГИТ имеет ограничение на количество репозиториев, потому выполняю тестовые задания в наиболее подходящем для хранения примеров кода.

- Для экономии времени задачу делаю максимально просто. Тем не менее, со своими важными пометками.

- В современных проектах, в БД цену лучше делать DECIMAL(100,8) - это связано с тем, что все расчеты, где важна точность до 8-го знака после запятой (фиатные и криптовые), выполняются на стороне PHP/JS/других языков в специализированых библиотеках: bcmath, js-decimal и так далее.
В REST API суммы денег так же лучше передавать в типе STRING - часто при неточных настройках микросервисов числовые типы могут конвертироваться в научную нотацию или как-либо ещё с потерей точности или искажением.

- Для ускорения использовал процедурный стиль. Разумеется, всё надо делать с использованием composer/OOP/autoload/namespace.

- Так же raw JS - подход верный, но я бы использовал Vue/React - в зависимости от традиций проекта - они стандарт де-фато в построениях резиновых интерфейсов, форм.

# Поиск товаров с фильтрами (Исходная задача)

Реализация API для поиска товаров с фильтрацией, сортировкой и пагинацией на PHP, raw JavaScript и MySQL 8.

## Задача

Реализовать HTTP‑endpoint `GET /api/products` для получения списка товаров с возможностью:
* фильтрации по различным параметрам;
* сортировки;
* пагинации.

## Структура данных

### Поля товара

Каждый товар содержит следующие поля:
* `id` — уникальный идентификатор (INT, AUTO_INCREMENT, PRIMARY KEY);
* `name` — название (VARCHAR 255, с FULLTEXT‑индексом для поиска);
* `price` — цена (DECIMAL 10,2);
* `category_id` — ID категории (INT, FOREIGN KEY на таблицу `categories`);
* `in_stock` — наличие на складе (BOOLEAN);
* `rating` — рейтинг (FLOAT, диапазон 0–5);
* `created_at` — дата создания (TIMESTAMP);
* `updated_at` — дата обновления (TIMESTAMP).

### Таблица категорий (`categories`)

* `id` — уникальный идентификатор;
* `name` — название категории;
* `created_at`, `updated_at` — временные метки.
