Here’s a detailed guide for **Wayland + KDE (KWin)** on how to detect the text cursor (caret) position and move the mouse cursor to it.

## Key limitations in Wayland

* **No direct access** to other apps’ UI elements (unlike X11).
* **Security sandbox**: Apps can’t freely read each other’s UI data.
* **Accessibility required**: You need explicit permissions.

## Solution overview

The most reliable approach combines:
1. **Accessibility APIs** (AT‑SPI2) to get caret position.
2. **KWin scripting** or **DBus** to get/set mouse position.
3. **ydotool** for mouse movement (Wayland‑compatible).

---

## Step‑by‑step implementation

### Step 1. Enable accessibility

1. Open **System Settings** → **Accessibility** → **Screen Reader**.
2. Enable **Screen Reader** (activates AT‑SPI2).
3. Alternatively, run:
   ```bash
   gsettings set org.gnome.desktop.a11y.applications screen-reader-enabled true
   ```

### Step 2. Detect text cursor position

#### Method A. Python + AT‑SPI2 (recommended)

Install dependencies:
```bash
sudo apt install python3-gi gir1.2-atspi-2.0  # Debian/Ubuntu
sudo dnf install python3-gobject at-spi2-core          # Fedora
```

Python script (`get_caret_wayland.py`):
```python
#!/usr/bin/env python3
import dbus
from dbus.mainloop.glib import DBusGMainLoop
from gi.repository import GLib

DBusGMainLoop(set_as_default=True)
bus = dbus.SessionBus()

def get_caret_position():
    try:
        # Get root accessible object
        root = bus.get_object('org.a11y.atspi.Registry', '/org/a11y/atspi/accessible/root')
        registry_iface = dbus.Interface(root, 'org.a11y.atspi.Accessible')

        # Navigate to focused component
        app = registry_iface.GetApplication()
        desktop = app.GetDesktop(0)
        active_window = desktop.GetActiveWindow()
        focused = active_window.GetFocusedComponent()

        # Check if it's a text component
        text_iface = dbus.Interface(focused, 'org.a11y.atspi.Text')
        caret_offset = text_iface.GetCaretOffset()
        rect = text_iface.GetCharacterExtents(caret_offset, 0)  # (x, y, width, height)

        # Return center of caret rectangle
        x_center = rect[0] + rect[2] // 2
        y_center = rect[1] + rect[3] // 2
        return x_center, y_center
    except Exception as e:
        print(f"Error detecting caret: {e}")
        return None

pos = get_caret_position()
if pos:
    print(f"{pos[0]},{pos[1]}")
else:
    print("caret,not_found")
```

#### Method B. KWin script (mouse position only)

To get the **current mouse position** (not text caret), use a KWin script:

1. Create `get_mouse.js`:
   ```javascript
   console.log(`x=${workspace.cursorPos.x},y=${workspace.cursorPos.y}`);
   ```
2. Run via DBus:
   ```bash
   SCRIPT=$(qdbus org.kde.KWin /Scripting org.kde.kwin.Scripting.loadScript /path/to/get_mouse.js)
   qdbus org.kde.KWin /Scripting/Script${SCRIPT} org.kde.kwin.Script.run
   qdbus org.kde.KWin /Scripting/Script${SCRIPT} org.kde.kwin.Script.stop
   ```

### Step 3. Move mouse cursor

Use **ydotool** (Wayland equivalent of `xdotool`):

1. Install `ydotool`:
   ```bash
   # From source
   git clone https://github.com/ReimuNotMoe/ydotool
   cd ydotool && sudo make install
   ```
2. Move mouse to coordinates:
   ```bash
   ydotool mousemove --absolute --x $x --y $y
   ```

### Step 4. Complete script

Combine all steps:
```bash
#!/bin/bash

# Get caret position
coords=$(python3 /path/to/get_caret_wayland.py 2>/dev/null)

if [[ $coords == "caret,not_found" ]] || [[ -z $coords ]]; then
    echo "Error: Could not detect text cursor."
    exit 1
fi

IFS=',' read -r target_x target_y <<< "$coords"

# Move mouse (Wayland)
ydotool mousemove --absolute --x "$target_x" --y "$target_y"

echo "Mouse moved to text caret at ($target_x, $target_y)."
```

## Alternative approaches

### Option 1. Remote Desktop portal (libei)

For production tools, use the **Wayland Remote Desktop portal** via DBus. This requires:
* User granting accessibility permissions.
* Using libraries like **libei** (Emulated Input).

### Option 2. uinput (root required)

If you have root access:
1. Use `uinput` to simulate mouse movement.
2. Combine with AT‑SPI2 for caret detection.
3. Note: This bypasses Wayland security — not recommended for regular use.

## Troubleshooting

* **App doesn’t expose caret**: Only GTK/Qt apps with accessibility support work. Terminals (e.g., Konsole) may not expose caret via AT‑SPI.
* **Permission denied**: Ensure user is in the `input` group (`sudo usermod -a -G input $USER`).
* **ydotool not found**: Install from source or check your distro’s package manager.
* **KWin script errors**: Verify KWin scripting is enabled in **System Settings** → **Window Management** → **KWin Scripts**.
